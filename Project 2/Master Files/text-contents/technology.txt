<h2>Technology overview</h2>
      <p>The sometimes complex and continually evolving technology infrastructure of Web 2.0 includes <a href="http://en.wikipedia.org/wiki/Computer_server" title="Computer server">server</a>-software, <a href="http://en.wikipedia.org/wiki/Content_syndication" title="Content syndication">content-syndication</a>, <a href="http://en.wikipedia.org/wiki/List_of_network_protocols" title="List of network protocols">messaging-protocols</a>, standards-oriented <a href="http://en.wikipedia.org/wiki/Web_browser" title="Web browser">browsers</a> with <a href="http://en.wikipedia.org/wiki/Plugin" title="Plugin">plugins</a> and <a href="http://en.wikipedia.org/wiki/Extension" title="Extension">extensions</a>, and various client-applications. The differing, yet complementary approaches of such elements provide Web 2.0 sites with <a href="http://en.wikipedia.org/wiki/Computer_data_storage" title="Computer data storage">information-storage</a>,  creation, and dissemination challenges and capabilities that go beyond  what the public formerly expected in the environment of the so-called  &quot;Web 1.0&quot;.</p>
      <p>Web 2.0 websites typically include some of the following features/techniques:</p>
      <ul>
        <li><a href="http://en.wikipedia.org/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">Cascading Style Sheets</a> to aid in the separation of presentation and content</li>
        <li><a href="http://en.wikipedia.org/wiki/Folksonomy" title="Folksonomy">Folksonomies</a> (collaborative tagging, social classification, social indexing, and social tagging)</li>
        <li><a href="http://en.wikipedia.org/wiki/Microformats" title="Microformats">Microformats</a> extending pages with additional <a href="http://en.wikipedia.org/wiki/Semantics" title="Semantics">semantics</a></li>
        <li><a href="http://en.wikipedia.org/wiki/Representational_State_Transfer" title="Representational State Transfer">REST</a> and/or <a href="http://en.wikipedia.org/wiki/Extensible_Markup_Language" title="Extensible Markup Language">XML</a>- and/or <a href="http://en.wikipedia.org/wiki/JSON" title="JSON">JSON</a>-based <a href="http://en.wikipedia.org/wiki/Application_programming_interface" title="Application programming interface">APIs</a></li>
        <li><a href="http://en.wikipedia.org/wiki/Rich_Internet_application" title="Rich Internet application">Rich Internet application</a> techniques, often <a href="http://en.wikipedia.org/wiki/Ajax_%28programming%29" title="Ajax (programming)">Ajax</a>-based</li>
        <li>Semantically valid <a href="http://en.wikipedia.org/wiki/XHTML" title="XHTML">XHTML</a> and <a href="http://en.wikipedia.org/wiki/HTML" title="HTML">HTML</a> <a href="http://en.wikipedia.org/wiki/Markup_language" title="Markup language">markup</a></li>
        <li>Syndication, aggregation and notification of data in <a href="http://en.wikipedia.org/wiki/RSS_%28file_format%29" title="RSS (file format)">RSS</a> or <a href="http://en.wikipedia.org/wiki/Atom_%28standard%29" title="Atom (standard)">Atom</a> feeds</li>
        <li><a href="http://en.wikipedia.org/wiki/Mashup_%28web_application_hybrid%29" title="Mashup (web application hybrid)">mashups</a>, merging content from different sources, client- and server-side</li>
        <li><a href="http://en.wikipedia.org/wiki/Weblog" title="Weblog">Weblog</a>-publishing tools</li>
        <li><a href="http://en.wikipedia.org/wiki/Wiki" title="Wiki">wiki</a> or <a href="http://en.wikipedia.org/wiki/Internet_forum" title="Internet forum">forum</a> software, etc., to support <a href="http://en.wikipedia.org/wiki/User_generated_content" title="User generated content">user-generated content</a></li>
      </ul>
